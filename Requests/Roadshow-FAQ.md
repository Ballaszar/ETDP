# Roadshow FAQ

## Is this fully automated with no human control?
No. The platform automates repeatable workflow steps, but operators remain in control of qualification selection, imports, quality checks, and final exports.

## How do you ensure compliance quality?
Workflow sequence and prerequisite guards enforce order. Content and exports follow mapped structures and review points before final publication.

## Can it work in low-connectivity or offline-first environments?
Yes. Local-first operation is supported, with cloud-dependent routes used only when runtime configuration allows them.

## What if we import the wrong template?
Use controlled re-import patterns and correction flows. Replace modes are available on specific routes for clean resets.

## How do you prevent cross-qualification contamination?
Qualification context is central. Every major capture, import, and export route is scoped to selected qualification identifiers.

## What if users are blocked by workflow prerequisites?
Use the warning card guidance, navigate to the missing step, complete minimum required records, then recheck workflow.

## Is there diagnostic support?
Yes. Use `/system-diagnostics` and relevant API diagnostics routes to capture failure context and correlation IDs.

## What are the highest-value outputs for institutions?
Learner guide, workbook, slide decks, learning schedules, and structured lesson-plan artifacts tied to qualification context.

## Can Mira explain and orchestrate the workflow end-to-end?
Yes. Mira is configured as an in-app workflow orchestrator with qualification-aware knowledge context and character blueprint controls.
